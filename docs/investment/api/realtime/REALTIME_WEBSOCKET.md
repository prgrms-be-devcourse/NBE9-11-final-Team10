# KIS 국내주식 실시간호가 (KRX)

## 개요

국내 주식의 실시간 호가창 데이터를 WebSocket으로 수신한다.

- API ID : 실시간-004
- TR_ID : H0STASP0
- 통신 방식 : WebSocket
- 대상 : KRX 국내 주식

---

## 사용 목적

모의투자 서비스에서 다음 기능 구현 시 활용한다.

- 실시간 호가창 표시
- 매수/매도 주문 가격 판단
- 지정가 주문 시뮬레이션
- 매수/매도 잔량 분석
- 체결 가능 여부 판단

## 요청 데이터

### Header

| 필드           | 설명           |
|--------------|--------------|
| approval_key | 웹소켓 접속키      |
| custtype     | P(개인), B(법인) |
| tr_type      | 1(등록), 2(해제) |
| content-type | utf-8        |

### Body

| 필드     | 설명               |
|--------|------------------|
| tr_id  | H0STASP0         |
| tr_key | 종목코드 (예: 005930) |

## 응답 예시

실제 응답은 ^ 문자로 구분된 문자열 형태로 전달된다.

text 종목코드 ^ 영업시간 ^ 시간구분 ^ 매도호가1 ^ 매도호가2 ... ^ 매수호가1 ^ 매수호가2 ...

## 우리 서비스 적용 방식

text KIS WebSocket ↓ 실시간 호가 수신 ↓ Spring WebSocket Manager ↓ 호가 DTO 변환 ↓ SSE 전송 ↓ Frontend

서버 측 멀티 인스턴스 환경에서의 WebSocket/SSE/Redis 연동 설계는
[실시간 호가 스트리밍 서버 설계](../../streaming/REALTIME_ORDERBOOK_STREAMING_DESIGN.md)를 따른다.

## 모의투자 주문 처리

### 시장가 매수

text 사용자 시장가 매수 ↓ 매도 1호가 가격으로 체결

### 시장가 매도

text 사용자 시장가 매도 ↓ 매수 1호가 가격으로 체결

### 지정가 주문

text 사용자 : 71,500원 매수 주문 ↓ 현재 매도 1호가 : 71,800원 ↓ 체결 대기 ↓ 매도 호가가 71,500원 도달 ↓ 체결 처리

## MVP 구현 범위

### 1차

- 실시간 호가 조회
- 시장가 매수/매도

- 지정가 주문

- 체결가 API(H0STCNT0) 추가 연동
- 체결 내역 표시
- 캔들/차트 데이터 연동

참고로 실제 증권사처럼 주문 체결을 흉내내려면 체결가(H0STCNT0)보다 오히려 호가(H0STASP0)가 더 중요해.  
사용자가 "71,500원에 매수" 같은 지정가 주문을 넣을 수 있게 하려면 결국 현재 호가창을 기준으로 체결 여부를 판단해야 하거든.

그래서 네 프로젝트 방향이 "모의 투자" 라면:

- MVP: H0STASP0 (실시간 호가)
- 고도화: `H0STASP0 + H0STCNT0`

조합이 가장 자연스럽다.
