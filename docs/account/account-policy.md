# Account 도메인 정책 요약

팀원이 Account 정책을 빠르게 확인하기 위한 문서다. 자세한 구현 흐름은 `account-domain.md`를 함께 본다.

## 1. 도메인 책임

Account 도메인은 **계좌 자체의 정보와 실제 잔액**을 관리한다.

| 책임 | 설명 |
| --- | --- |
| 계좌 생성 | 본인인증 완료 사용자의 입출금계좌 생성 |
| 계좌 조회 | 사용 중 계좌, 해지 계좌, 계좌 상세 조회 |
| 계좌 별칭 | 활성 계좌의 별칭 수정 |
| 계좌 비밀번호 | 생성, 설정, 변경, 검증 |
| 계좌 해지 | 조건을 만족한 일반 계좌를 `CLOSED` 처리 |
| 잔액 변경 | `deposit()`, `withdraw()` 메서드로만 처리 |

## 2. 계좌 타입 정책

현재 일반 계좌 개설 API에서 사용자가 직접 만들 수 있는 계좌 타입은 **`DEPOSIT`만** 허용한다.

| AccountType | 의미 | 생성 위치 | 일반 입금 | 일반 송금 보내기 | 일반 송금 받기 |
| --- | --- | --- | --- | --- | --- |
| `DEPOSIT` | 입출금계좌 | Account 계좌 개설 API | 가능 | 가능 | 가능 |
| `SAVING_DEPOSIT` | 예금 전용 계좌 | Saving 예금 가입 로직 | 불가 | 불가 | 불가 |
| `SAVING_INSTALLMENT` | 적금 전용 계좌 | Saving 적금 가입 로직 | 불가 | 불가 | 불가 |

주의:

- `SAVINGS`, `FOREIGN_CURRENCY`, `INVESTMENT`는 AccountType에 추가하지 않는다.
- Saving, Exchange/ForeignWallet, Investment는 별도 도메인이다.
- 예/적금 전용 계좌는 Account로 저장하지만 자유 입출금 계좌가 아니다.

## 3. 계좌 상태 정책

| AccountStatus | 의미 | 일반 목록 조회 | 해지 목록 조회 |
| --- | --- | --- | --- |
| `ACTIVE` | 정상 사용 가능 | 포함 | 제외 |
| `SUSPENDED` | 정지 상태 | 포함 | 제외 |
| `CLOSED` | 해지/종료 상태 | 제외 | 포함 |

계좌 상세 조회는 본인 소유 계좌면 상태와 관계없이 조회 가능하다.

## 4. 계좌 생성 정책

입출금계좌 생성 조건:

```text
1. 요청 사용자가 존재해야 한다.
2. 사용자의 본인인증이 완료되어 있어야 한다.
3. 요청 accountType은 DEPOSIT이어야 한다.
4. 계좌 비밀번호는 숫자 6자리여야 한다.
5. 계좌번호는 서버에서 생성하고 중복되면 재시도한다.
6. 계좌 비밀번호는 해시로 저장한다.
7. 초기 잔액 0원, 상태 ACTIVE로 저장한다.
```

주요 실패 상황:

| 상황 | ErrorCode |
| --- | --- |
| 사용자 없음 | `USER_NOT_FOUND` |
| 본인인증 미완료 | `IDENTITY_VERIFICATION_REQUIRED` |
| `DEPOSIT` 외 타입 요청 | `INVALID_ACCOUNT_TYPE` |
| 계좌번호 생성 실패 | `ACCOUNT_NUMBER_GENERATION_FAILED` |

## 5. 계좌 비밀번호 정책

| 기능 | 정책 |
| --- | --- |
| 계좌 생성 | 생성 요청에 비밀번호 필수 |
| 비밀번호 형식 | 숫자 6자리 |
| 저장 방식 | 원문 저장 금지, 해시 저장 |
| 비밀번호 설정 | 기존 계좌 중 비밀번호가 없는 활성 계좌만 가능 |
| 비밀번호 변경 | 현재 비밀번호 검증 후 새 비밀번호 저장 |
| 같은 비밀번호 변경 | 현재 비밀번호와 새 비밀번호가 같으면 실패 |
| 송금/예적금 가입/계좌 해지 | 출금 또는 상태 변경 전 계좌 비밀번호 검증 |

주요 실패 상황:

| 상황 | ErrorCode |
| --- | --- |
| 비밀번호 미설정 | `ACCOUNT_PASSWORD_NOT_SET` |
| 이미 비밀번호 설정됨 | `ACCOUNT_PASSWORD_ALREADY_SET` |
| 비밀번호 불일치 | `ACCOUNT_PASSWORD_MISMATCH` |
| 현재 비밀번호와 새 비밀번호 동일 | `ACCOUNT_PASSWORD_SAME` |

## 6. 계좌 해지 정책

일반 계좌 해지는 민감 작업이므로 계좌 비밀번호를 검증한다.

```text
1. 본인 소유 계좌를 락 조회한다.
2. 계좌 상태가 ACTIVE인지 확인한다.
3. 잔액이 0원인지 확인한다.
4. 계좌 비밀번호가 일치하는지 확인한다.
5. 계좌 상태를 CLOSED로 변경한다.
```

주의:

- 잔액이 남아 있으면 해지할 수 없다.
- 일반 계좌 해지는 Account 도메인에서 처리한다.
- 예/적금 전용 계좌 해지는 Saving 중도해지/만기 처리에서만 수행한다.

## 7. 잔액 변경 정책

계좌 잔액은 직접 필드 수정하지 않는다.

| 메서드 | 설명 |
| --- | --- |
| `deposit(amount)` | 잔액 증가 |
| `withdraw(amount)` | 잔액 부족 검증 후 잔액 감소 |

원칙:

- 금액 변경은 서비스 트랜잭션 안에서 처리한다.
- 금액 변경 대상 계좌는 DB 락 조회를 우선 사용한다.
- 거래내역의 `balanceAfter`는 해당 계좌 기준 거래 후 잔액이다.

## 8. 락 정책

동시에 같은 계좌 잔액이 변경되는 상황을 막기 위해 DB 락을 사용한다.

| Repository 메서드 | 사용 목적 |
| --- | --- |
| `findByIdForUpdate` | 계좌 ID 기준 명시적 락 |
| `findByIdAndUserIdForUpdate` | 본인 소유 계좌 조회 + 락 |

두 계좌가 함께 바뀌는 경우:

```text
계좌 ID가 작은 계좌부터 락을 잡는다.
```

이유:

- 서버가 여러 대여도 같은 순서로 락을 잡게 한다.
- 서로 다른 순서로 락을 잡아 생기는 데드락 위험을 줄인다.

## 9. 다른 도메인과의 경계

| 도메인 | Account와의 관계 |
| --- | --- |
| Transfer | 일반 입금/송금 시 `DEPOSIT` 계좌 잔액 변경 |
| Saving | 예/적금 가입, 자동이체, 해지, 만기 시 전용 Account 잔액 변경 |
| Transaction | Account 기준 거래내역 저장 |
| Exchange/ForeignWallet | 별도 도메인, AccountType에 외화 타입 추가하지 않음 |
| Investment | 별도 도메인, AccountType에 투자 타입 추가하지 않음 |

## 10. 팀원 확인 체크리스트

Account 관련 작업 전 아래를 확인한다.

- [ ] 일반 계좌 개설 API에서 `DEPOSIT`만 허용하는가?
- [ ] 계좌 비밀번호는 숫자 6자리로 검증하는가?
- [ ] 계좌 비밀번호 원문을 저장하지 않는가?
- [ ] 금액 변경 전 계좌를 락 조회하는가?
- [ ] 잔액은 `deposit()`, `withdraw()`로만 변경하는가?
- [ ] 예/적금 전용 계좌를 일반 입금/송금에 사용하지 않는가?
- [ ] 계좌 해지 시 ACTIVE, 잔액 0원, 비밀번호 검증을 모두 하는가?
